## Project Overview

A minimalistic cinematography portfolio website inspired by ligthelm.work. Static site featuring video grid with hover previews, category filtering, and full-screen Vimeo playback.

## Architecture

### Pages
- `index.html` - Main portfolio grid with video projects (links to individual project pages)
- `photography.html` - Photography gallery page
- `projects/*.html` - 31 individual project pages with embedded Vimeo players
- All pages share the same header/information overlay pattern

### Key Components

**Video System (Link-Based Navigation)**:
- **Preview layer**: Local video files in `media/previews/` (5-second clips, 640px wide, autoplay on hover)
- **Full playback**: Individual project pages with embedded Vimeo iframes
- Each project card on index.html is an `<a>` tag linking to `projects/project-slug.html`
- Each project page includes video player, project details, and prev/next navigation
- No modal overlay system - clean page-to-page navigation

**Project Cards**:
- Grid layout defined in HTML with inline styles for thumbnails
- Each card is an `<a href="projects/slug.html">` link
- `data-category` attribute used for filtering
- Can use `.large` class to span 2x2 grid spaces
- Photography card uses special handling to link to photography.html

**State Management**:
- Pure vanilla JavaScript, no frameworks
- `projects` array in script.js contains project metadata for grid display
- Each project page contains its own embedded data
- Filters update DOM visibility and URL hash for category navigation

## Common Commands

### Local Development
```bash
# Serve locally (Python 3)
python -m http.server 8000

# Or with Node.js
npx http-server
```

### Video Processing
```bash
# Generate preview clips from source videos
./create_previews.sh
```

The script:
- Processes all videos in `~/Documents/website-source-videos/`
- Creates 5-second clips at 640px width
- Outputs to `media/previews/`
- Uses ffmpeg with fast preset, CRF 28, strips audio

## Adding New Projects

1. **Add source video** to `~/Documents/website-source-videos/`
2. **Generate preview**: Run `./create_previews.sh`
3. **Add thumbnail** JPG to `media/thumbnails/`
4. **Update generation script**: Add project data to `scripts/update-all-v2-pages.py`
5. **Generate project page**: Run the script to create `projects/project-slug.html`
6. **Update `index.html`**: Add new project card as `<a href="projects/slug.html">` with:
   - `data-category` for filtering
   - Inline style with thumbnail URL
   - `<video>` tag pointing to preview file
7. **Update `script.js`**: Add entry to `projects` array for grid display and navigation

## Styling

- Pure CSS, no preprocessors
- Dark theme: `#000` background, `#fff` text
- `mix-blend-mode: difference` on header for contrast over any background
- Responsive breakpoints in `style.css` adjust grid columns
- `photography-style.css` contains photography page-specific styles

## Important Implementation Details

- All project cards are `<a>` tags linking to individual project pages
- Photography card links to photography.html instead of a project page
- Each project page has prev/next navigation with category-aware filtering
- Info page shares markup between HTML files but has duplicate JavaScript
- No build step or bundling required - all assets served directly
- Generation scripts archived in `scripts/` directory for maintenance

## Video Preview Specs

All preview clips are optimized for web:
- **Duration**: 5 seconds
- **Resolution**: 640px width (maintains aspect ratio)
- **Codec**: H.264 (libx264)
- **Preset**: fast
- **CRF**: 28 (balanced quality/size)
- **Audio**: Removed (smaller files, muted anyway)
- **File sizes**: 41KB - 425KB (avg ~165KB)
- **Total preview folder**: 2.5MB (vs 8.4GB for full videos)
